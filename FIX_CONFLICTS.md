# Как исправить конфликты слияния

## Ситуация:
У тебя есть неразрешенные конфликты после `git merge danial-913534`

## Решение:

### Шаг 1: Проверь, какие файлы в конфликте

В терминале выполни:
```bash
git status
```

Ты увидишь файлы с пометкой "both modified" или "unmerged"

### Шаг 2: Открой файлы с конфликтами

Конфликты выглядят так:
```
<<<<<<< HEAD
Твой код из main
=======
Код из danial-913534
>>>>>>> danial-913534
```

### Шаг 3: Разреши конфликты

**Вариант А: Оставить версию из main**
- Удали все маркеры конфликта
- Оставь только код между `<<<<<<< HEAD` и `=======`

**Вариант Б: Оставить версию из danial-913534**
- Удали все маркеры конфликта
- Оставь только код между `=======` и `>>>>>>> danial-913534`

**Вариант В: Объединить обе версии**
- Удали маркеры конфликта
- Оставь нужные части из обеих версий

### Шаг 4: После исправления всех конфликтов

```bash
# Добавь исправленные файлы
git add .

# Заверши merge
git commit -m "Merge branch 'danial-913534' into main"
```
